Two-way data binding

In Angular 1, the default option was two-way data binding.
Thought, it 's easy to implement it cause a lots of problems (cascading effects, coupling of "data layout tree" and "dom view tree", etc.)
The recommended way in Angular 2, and in many modern web frameworks (such as React), is to adopt a pattern of one-way data binding

    Event binding

    Allows to react to user interactions
    We can send information from the view to the component class.
    target_event - usually, a native DOM element event, but you can also use a custom events.
    event_handler - the Component Method to handle the event.

    Event Target

    To capture the event object, pass $event as parameter in the event_handler
    
			
		
    
			inputEcho:string;
			// ...
			onInput(e){
				this.inputEcho = e.target.value;
			}

    Two-way Data Binding

    Both display a data property and update that property when the user makes changes.
    The two-way binding syntax is just syntactic sugar for a property binding and an event binding
    When using Two-way Data Binding for form controls is easier to use the [(ngModel)] attribute directive
    Note, FormsModule is required to use ngModel
    
			import { FormsModule } from '@angular/forms';

			@NgModule({
				imports: [
					BrowserModule,
					FormsModule
