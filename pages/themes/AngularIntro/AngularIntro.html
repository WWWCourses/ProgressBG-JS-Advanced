Intro

How it started?

2009: "GetAngular" a side project of Misko Hevery (working for Google) and Adam Abrons.
The idea: an end-to-end tool that would allowed web designers to interact easily with both the frontend and the backend.
Misko Hevery used GetAngular in an internal Google project, and short after Google commence sponsoring the project.
Soon, AngularJs was released as open source.
Angular is still backed up by Google.

AngularJS ≠ Angular

AngularJS is a front-end web application framework!

Angular is a platform that allows you to develop fast and scalable applications for web, mobile web, native mobile and even native desktop.

Versions and time-line

AngularJS
Initial release: v1.0.0 / October 20, 2010.
Current release: Version Support Status @angularjs
Angular
A complete rewrite of AngularJS!
Initial release: v.2.0.0 / September 14, 2016
Current release: Angular Releases @angular

Angular + TypeScript

Angular 2.0 is build entirely in TypeScript, and as a result, using TypeScript with Angular provides a seamless experience.
Though you can use plain JavaScript for your Angular App, the preferred way is to use TypeScript

Angular: Why TypeScript?

TypeScript

TypeScript is an open-source language, backed up by Microsoft
It is a typed superset of JavaScript that compiles to plain JavaScript.
TypeScript provides optional static typing, classes and interfaces - i.e. it provides class-based OOP for JavaScript.
TypeScript supports new ECMAScript standards and compiles them to older ECMAScript versions
Enhanced IDE/Editor support

ng completions - optional

Auto-completions for ng commands. To install in Windows:


			# appends the completions at the end of bash-profile
			ng completion --bash |cat >> ~/.bash_profile

			# activate completions immediately, or just restart the terminal
			source ~/.bash_profile

Now when you start typing ng something in you terminal, and hit tab, the command will be auto-completed

Get familiar with Angular CLI

Overview

The Angular CLI is a command-line interface tool that you use to initialize, develop, scaffold, and maintain Angular applications
Reference: angular.io/cli

      # Create, build and run(serve with live-reload) an Angular Project:
      ng new MyApp
      cd MyApp
      ng serve --open

Main commands

CommandAction
helpShow help
newCreates a new directory and a new Angular app.
updateUpdates your application and its dependencies.
serveBuilds and serves your app, rebuilding on file changes.
buildBuilds your app and places it into the output path (dist/ by default).
generateGenerates and/or modifies files based on a schematic.
addAdd support for a library to your project.
testRun unit tests in existing project.
e2eRun e2e tests in existing project.
lintLints code in existing project.

Create the HelloWorld project in 5 minutes

Create the HelloWorld project in 5 minutes

Prerequisites

Make sure, you can open your Terminal and navigate into the folder where you want the Angular Project to be created.
For VSCode users, check: Integrated Terminal docs.


      # create the project
      ng new HelloWorld

      # go to project root folder
      cd HelloWorld

      # run the HelloWorld, using local server
      ng serve --open
the --open option instructs the serve to open the URL at which the project will be served. The default is : http://localhost:4200/

Project Directory Structure and Files

Project Directory Structure and Files

ng new creates the following structure:

angular.json (.angular-cli.json in angular cli versions < 6.0)
configuration for Angular CLI
e2e/
for end-to-end (e2e) tests
node_modules/
here are installed project's node modules dependencies
package.json
dependencies and devDependencies sections specifies the node modules dependencies
package-lock.json
it is automatically generated for any operations where npm modifies either the node_modules tree, or package.json
src/app/
here is the actual logic of your app
tsconfig.json
TypeScript compiler configuration
tslint.json
tslint configuration

Different Angular version on same machine?

Different Angular version on same machine?

You just need to have angular/cli installed globally! Make sure you have the latest version:
npm install -g @angular/cli@latest
The Angular project installed on a machine use their local version of angular. Check by that going to an Angular project and:

			ng version
			Your global Angular CLI version (7.1.0) is greater than your local
			version (1.7.4). The local Angular CLI version is used.
